# ITD-01

### Лабораторная №1: Создание интегральной схемы в представлении JSON.

### Цель работы: 
Освоить графовое представление топологии интегральных схем для последующей верификации DRC с помощью GNN.

### Задача:

  1. Создать свой JSON-файл топологии в соответствии со описанной структурой и требованиями.
  2. Загрузить созданный файл в визуализатор топологий интегральных схем (visualiz_json.py).
  3. Сделать скриншот полученной топологии.
  4. Сформировать отчёт.

### Требования к JSON-файлу
| Параметр                          | Значение          | 
| --------------------------------- | ----------------- | 
| Количество элементов (element_id) | >10               | 
| Нарушения DRC                     | ≥2                | 
| Минимальный допустимый зазор      | 2.0               | 

### Структура JSON-файла.

Структура входного JSON-файла четко регламентирована. Файл состоит из двух ключевых полей nodes и edges.

Поле nodes представляет собой список (массив) объектов, где каждый объект описывает один узел графа. Для каждого узла обязательны следующие атрибуты: 
1) id - уникальный целочисленный идентификатор узла. 
2) x, y - целочисленные значения координат узла в двумерном пространстве.  
3) element_id - целочисленный идентификатор элемента, которому принадлежит данный узел. Этот признак позволит модели понимать, какие узлы относятся к одному и тому же физическому объекту.

### 
```json
"nodes": [
  {
    "id": 1,                    //  Уникальный ID (int)
    "x": 50,                    //  Координата X (int)
    "y": 30,                    //  Координата Y (int)  
    "element_id": 1             //  ID элемента (int)
  }
]
```

Поле edges является списком объектов, каждый из которых описывает одно ребро. Структура объекта ребра зависит от его типа: 
1) source, target - целочисленные идентификаторы исходного и целевого узлов, которые соединяет ребро. 
2) Type - строковый категориальный признак, определяющий тип связи. Он может принимать два значения: counter для контурных ребер, и spatial для пространственных. 
3) distance - числовое значение, представляющее точное евклидово расстояние между соединенными узлами. Этот атрибут 
присутствует только у ребер типа spatial.

### 
```json
"edges": [
  {
    "source": 1,               //  Узел-источник (int)
    "target": 2,               //  Узел-цель (int)
    "type": "contour",         //  "contour" (контурный)
    "distance": null           //  НЕТ для contour
  },
  {
    "source": 5, 
    "target": 12,
    "type": "spatial",         //  "spatial" (пространственный)
    "distance": 1.85           //  Евклидово расстояние
  }
]
```
